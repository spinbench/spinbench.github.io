PDDL: Planning Domain Definition Language

PDDL Introduction

Name: Planning Domain Definition Language (PDDL)

Type: Classical Planning Tasks

Description: A formal language used to define planning problems where an agent must find a sequence of actions to reach a goal state from an initial state.

Purpose: Tests core reasoning skills including factual retrieval, spatial reasoning, and multi-step planning with progressive complexity levels.

Benchmark Settings

Test Categories: Three subcategories to test different aspects of reasoning:

    1. Factual Retrieval: Tasks in domains like elevator, grid, and floortile with progressively expanding state spaces
    2. Spatial Reasoning: Modified floortile scenario challenging models to track agent location from relative instructions
    3. Full Classical Planning: 21 domains (1,280 tasks) involving sequential decisions, resource management, and constraint handling

Evaluation Metrics:

    1. Accuracy: Percentage of correct solutions
    2. N-Step Look Ahead: Weighted measure highlighting performance on complex problems
    3. Error Analysis: Breaking constraints, goal not satisfied, and other errors

Models Evaluated: o1, o1-mini, o3-mini, GPT-4o, GPT-4-turbo, Claude 3.5 Sonnet, Claude 3.5 Haiku, DeepSeek-R1, Llama3.3-70b
